Introduction¶

Our objective in this activity is to create phylogenetic trees and to make different queries about them, such as knowing which is the common ancestor, finding out the path from one leaf to another, or simply counting how many terminals are thereñ

To do this, first of all, we will look for examples of proteins that are relevant because of their functions or applications and carry out the study on them. When we have their sequences, we will align them to find out the degree of similarity and, finally, we will create the phylogenetic trees on which we will work.

Using these phylogenetic trees, we'll use some of the functions included in the 'Phylo' section in BioPython to showcase the way they work and their many uses.

Protein search and reading¶

On the search for proteins, we considered it appropriate to select two different ones:

- Beta casein: it is a major protein component of milk and, in conjunction with the other caseins, it is assembled into micelles. The casein micelles determine many of the physical characteristics of milk, which are important for stability during storage and for milk-processing properties. 
- Pyrimidine/purine Nucleoside Phosphorylase: it is a protein that catalyzes the phosphorolysis of diverse nucleosides, yielding D-ribose 1-phosphate and the respective free bases. It can use uridine, adenosine, guanosine, cytidine, thymidine, inosine and xanthosine as substrates. It also catalyzes the reverse reactions.

For both cases, we have selected 10 organisms which have that protein. In the case of beta casein, these are organisms such as humans, gorillas and horses, among others; while in the second case, we have selected different species of bacteria.

def read_multiFasta(protein_file):
    fastas = {}
   
    file = open(protein_file)
   
    for line in file:
        
        if (line[0]==">"):
            head=line[1:]
            seq=""
            
        else:
            line=line.rstrip("\r")
            seq+=line.rstrip("\n")
            fastas[head]=seq
        
    return fastas

ClustalW2¶

In order to create the phylogenetic trees, we are going to use ClustalW2. ClustalW2 is the command-line interface of Clustal, a computer program for performing multiple sequence alignments and creating phylogenetic trees.

In our project, we will implement clustal_wrapper, a class which will use biopython functions of Alingment tools section, and whose purpose is to be an adapter for ClustalW2.

Download and install of ClustalW2¶

First, we will show the steps for downloading ClustalW2:

  1. Enter in Clustal W official website in: "http://www.clustal.org/clustal2/">

Then download cluatalw suited for your computer.

  1. Use the setup file to install ClustalW2. Pay attention to the installation location.

  1. Now, we will train the ClustarW2 application with 'opuntia.fasta', a fasta file with some DNA sequences.

After choosing 'Do complete multiple alignments now Slow/Accurate', the application generates two files: opuntia.aln, with the alignment, and opuntia.dnd, with the phylogenetic tree. We will save this last one as correctTree.dnd, for using it to test our python project results later.

Adapter class¶

Implementation¶

We want to use ClustalW2 from our python project to create phylogenetic trees. For this purpose, we use biopython functions in the class ClusterWrapper.

When we initialize clustal_wrapper, the only input parameter is the path where the file clustal.exe is. The function asks for it because we use it, with os package functions, instead of modifying the path. Then, the class has the function generate_aligment_tree, which needs the fasta file with the sequence and uses ClustalW2 to create the alignment and the phylogenetic tree. Subsequently, we will use this tree to show some of the functions included in the 'Phylo' section in BioPython.

import os
from Bio.Align.Applications import ClustalwCommandline


class ClustalWrapper:
    def __init__(self, clustal_exe_path):
        self.clustalw_exe = clustal_exe_path

    def generate_alignment_tree(self, fasta_file):
        try:
            clustalw_cline = ClustalwCommandline(self.clustalw_exe, infile=fasta_file, seqnos="ON")
            assert os.path.isfile(self.clustalw_exe), "Clustal W executable missing"
            stdout, stderr = clustalw_cline()
        except:
            raise Exception("ClustalW2 did not work")
Unit test¶

Although our class does not create the files itself, it makes use of ClustalW2 for generating the phylogenetic trees. Therefore, we have to check if the phylogenetic tree's file (file with dnd extension) is correct.

For that purpose, we create a unit test. We will use the phylogenetic tree generated from the ClustalW2 command-line using opuntia.fasta's file, called correctTree.dnd, and clustal_wrapper will generate another phylogenetic tree, opuntia.dnd, from the same multisequence file. Both trees should be equals.

Python has a package, filecmp, that allows us to compare two files using its function cmp and returning a boolean value which indicates if the two files are equals.

In [ ]:
import unittest
from phylo.clustal_wrapper import ClustalWrapper
import filecmp

class TestClustalWrapper(unittest.TestCase):

    def test_dndfile(self):
        clustalWPath = "D:\clustalForBiopython\CLustalW2.exe"
        clustalWrapper = ClustalWrapper(clustalWPath)
        clustalWrapper.generate_alignment_tree('..\\phylo\\proteins\\opuntia.fasta')
        isEqual = filecmp.cmp('..\\phylo\\test\\testFiles\\correctOpuntia.dnd', '..\\phylo\\proteins\\opuntia.dnd', shallow= False)
        self.assertTrue(isEqual)

if __name__ == '__main__':
    unittest.main()

Phylo¶

Finally, we use the Biopython functions intended to create and work with phylogenetic trees. There are a lot of them, and they allow us to work with the trees and understand them better.

Printing¶

Now, let's say we want to print our tree. Well, we have gour different options. We can print it using the clade format, we can print it drawing branches, we can create a new window with an image of it, or we can write in in xml format. We've created a function to make it easier to do this, with the different options ranging from 1 to 4.

In [4]:
def drawTree(tree, type):
    if type == 1:
        print(tree)
    elif type == 2:
        Phylo.draw_ascii(tree)
    elif type == 3:
        tree.rooted = True
        Phylo.draw(tree)
    elif type == 4:
        Phylo.write(tree, sys.stdout, "phyloxml")

More functions¶

In this section we'll explain a series of functions that can help us navigate and work with the trees in much more depth.

Common ancestors

One of the principal objectives of phylogenetic trees is to get to know the common ancestors between two species. Biopython's function to do so is common_ancestor. To be executed, it needs to know the name of the branches you are trying to connect. These names must be given like dictionaries, with the format {"name":"branchName"}. But once we try it with our files, we discover a problem.

Our tree has species with compound names, which have three parts and are separated by a |. This causes a problem with the method, since the common ancestor it returns isnt the real one. Here we have an example:

In [16]:
drawTree(treePPN, 1)

Looking at this tree, we can see the common ancestor between sp|Q9KKY0|PPNP_VIBCH and sp|A0KK62|PPNP_AERHH is a clade with branch_lenght=0.02779. But if we try to use the common_ancestor method, we get the wrong answer:

In [19]:
treePPN.common_ancestor({"name": "sp|Q9KKY0|PPNP_VIBCH"},{"name":"sp|A0KK62|PPNP_AERHH"})
Out[19]:
Clade(branch_length=0.07253, name='sp|P0C037|PPNP_ECOLI')

After a series of tests, we have discovered that the method only works when we have simpler names, so we have created alternative dnd files where we have changed each species' name, removing the first two words and sticking with the third one, which is always different. These files are BCSimp.dnd and PPNsimpl.dnd.

So let's try using them:

In [20]:
smpBC=Phylo.read("files/BCsimpl.dnd", "newick")
smpPPN=Phylo.read("files/PPNsimpl.dnd", "newick")

Now, we should get the correct results.

In [21]:
smpBC.common_ancestor({"name":"A0A0D9QY03_CHLSB"}, {"name":"CASB_HORSE"})
Out[21]:
Clade(branch_length=0.02017)
In [22]:
smpPPN.common_ancestor({"name":"PPNP_RALSO"}, {"name":"PPNP_CUPNH"})
Out[22]:
Clade(branch_length=0.07734)

These answers are indeed correct, as we can see if we draw the trees with the first drawing option. Since we have found this problem, from now on we'll use those new files with reduced names to avoid having more issues.

Now we have that sorted, we can explore the importance of the common_ancestor method. Maybe when we execute it and get an answer it isnt really clear, but there's something we can do to get a better perspective. We can color the path from this common ancestor to its heirs. This is very useful when we want to track the results of a mutation or a certain moment in evolution. There are two ways to do this:

First, if we don't know the exact location of the common ancestor we are looking for, we first use the common_ancestor function to get it, then we change the color of its ramifications, and lastly, we draw it.

In [23]:
comunBC=smpBC.common_ancestor({"name":"A0A0D9QY03_CHLSB"}, {"name":"CASB_HORSE"})
comunBC.color="#99ccff"
drawTree(smpBC, 3)
In [24]:
comunPPN=smpPPN.common_ancestor({"name":"PPNP_RALSO"}, {"name":"PPNP_CUPNH"})
comunPPN.color="#ffcccc"
drawTree(smpPPN, 3)

This second way doesn't require using the common_ancestor function, but it can only be done when you know the exact location of it beforehand. Therefore, it way may not be realistic when we have a bigger phylogenetic tree.

To represent the exact same common ancestors from before, we'd need to do the following (we have to first reload the files because the variables we have are already colored):

In [25]:
smpBC=Phylo.read("files/BCsimpl.dnd", "newick")
smpPPN=Phylo.read("files/PPNsimpl.dnd", "newick")
In [26]:

drawTree(smpBC,3)
In [27]:

drawTree(smpPPN,3)

Collapse

We begin with collapse, a method that, given an element from the tree, deletes it. If this element has any branches, they'll be connected to the elements' parent.

A variant of this method is collapse_all, which brings every leaf to the same level, deleting all ramifications

In [60]:
copPNN.collapse("PPNP_ECOLI")
drawTree(copPNN,3)
In [62]:
copPNN=copy.deepcopy(smpPPN)
copPNN.collapse_all()
drawTree(copPNN,3)

Ladderize

This method can organize a tree, showing the shallower branches first and the deeper ones last.

In [63]:
copPNN=copy.deepcopy(smpPPN)
copPNN.ladderize()
drawTree(copPNN,3)

Final thoughts¶

By creating this report we have further proved that bioinformatics are essential to biology, specially in fields related to genetics. The Phylo module in this Biopython package is really useful, providing a series of methods that can make the process of creating, navigating, consulting and modifiying a phylogenetic tree much easier.

It allows us to represent it in a really intuitive way, and the commands throughout which we can work with it are easy to use and remember. It includes a complete set of functions that, in our opinion, make a researcher's work easier, even if they aren't familiar with programming languages.
